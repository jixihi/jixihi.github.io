Linux学习第四天-文件目录类命令

Posted on 2026-04-09 In Linux学习

🐧 Linux 文件目录类基础命令

📁 一、 目录操作命令

1. 查看与切换目录

命令 功能说明 常用示例
pwd 打印当前工作目录的绝对路径 pwd
cd 切换工作目录 cd ../xxx（进入上级目录的 xxx）
cd -（返回上一次所在目录）

2. 列出目录内容 (ls)

参数 说明
ls 列出当前目录非隐藏内容
ls -a 显示所有文件（包含以 . 开头的隐藏文件）
ls -l 长格式显示（等效别名 ll）
ls -al 结合 -a 与 -l，完整显示所有文件详情

💡 ls -l 输出字段解析（从左到右）：
权限 硬链接数 所有者 所属组 文件大小 最后修改时间 文件/目录名

首字符为 d 表示目录，- 表示普通文件，l 表示软链接。

3. 创建与删除目录

命令 功能说明 注意事项
mkdir 创建目录 mkdir a / mkdir b c
嵌套创建需加 -p：mkdir -p d/e/f
rmdir 删除空目录 非空目录会报错。可配合 -p 递归删除空父目录链：rmdir -p d/e/f

2. 复制文件/目录 (cp)

  • -r：递归复制整个目录（复制文件夹必加）
  • 🛡️ 跳过覆盖确认：系统通常默认 cp 带 -i 参数会询问是否覆盖，可在命令前加反斜杠强制覆盖：\cp source dest
  • 路径行为逻辑：
    • 源/目标均为文件 → 复制并重命名
    • 源为文件，目标为目录 → 将文件复制到目标目录内
    • 源/目标均为目录 → 必须使用 -r 选项

3. 删除文件/目录 (rm)

命令 行为说明
rm a 删除文件 a（交互式确认）
rm -f a 强制删除，不弹出确认提示
rm -r a 递归删除目录 a（逐层询问是否删除）
rm -rf a 🔥 强制递归删除，不询问直接删除目录及内部所有内容
rm -f ./* 在目标目录内执行，清空当前目录所有文件/子目录（保留当前目录本身）

⚠️ 高危警告：rm -rf 威力极大且不可恢复，执行前务必 pwd 确认当前路径！

👀 三、 文件内容查看命令（补充）

在 Linux 中查看文本内容有三种核心命令，适用场景差异明显：

1. cat (Concatenate)

  • 特点：一次性将文件内容全部输出到终端，适合短小文件。
  • 常用用法：

2. more

  • 特点：分页查看器，只能向下翻页，适合快速浏览中等长度的日志。
  • 交互快捷键：
    • Space：向下翻一页
    • Enter：向下翻一行
    • q：退出查看
  • 用法：more app.log

3. less (⭐ 强烈推荐)

  • 特点：more 的现代增强版。支持双向翻页、实时搜索，且按需加载（不一次性读入内存），适合超大日志/数据文件。
  • 交互快捷键：
    • Space / PageDown：向下翻页
    • b / PageUp：向上翻页
    • /keyword：向下搜索关键词（按 n 跳至下一个，N 跳至上一个）
    • ?keyword：向上搜索
    • G：跳至文件末尾，g：跳至文件开头
    • q：退出
  • 用法：less huge_data.csv

📊 核心命令速查对比表

实际需求 推荐命令 备注
查看短配置/代码 cat -n 直观带行号
快速浏览长日志（仅往下翻） more 简单轻量
仔细翻阅大文件 / 需搜索 less ⭐ 内存友好，功能最全
创建/更新时间戳 touch 不修改内容
复制目录 cp -r 注意 -r 位置
强制覆盖不提示 \cp 绕过 alias cp='cp -i'
彻底删除目录 rm -rf ⚠️ 执行前三思
移动或重命名 mv 同盘移动极快（仅改元数据）

💡 实践建议
Linux 命令的学习重在肌肉记忆。建议在终端中搭配 man [命令] 或 [命令] --help 探索更多参数。遇到破坏性操作（尤其是 rm 和重定向 >）前，先 ls 确认环境，养成**“先查看，后执行”**的安全习惯。
