Linux学习第四天-文件目录类命令

Posted on 2026-04-09 Edited on 2026-04-10 In Linux学习

🐧 Linux 核心命令速查手册：文件、目录与终端管理

📁 一、 目录操作命令

命令 功能说明 常用示例 & 备注
pwd 打印当前工作目录的绝对路径 pwd
cd 切换工作目录 cd ../xxx（进入上级目录的 xxx）
cd -（返回上一次所在目录）
ls 列出目录内容 ls：默认非隐藏
ls -a：包含 . 开头的隐藏文件
ls -l：长格式详情（等效别名 ll）
ls -al：完整显示所有文件详情
mkdir 创建目录 mkdir dir
mkdir -p a/b/c：递归创建多级目录
rmdir 删除空目录 rmdir dir
rmdir -p a/b/c：递归删除空父目录链（遇非空即停止）

📌 ls -l 输出字段解析（从左至右）：
权限 硬链接数 所有者 所属组 文件大小 最后修改时间 名称
首字符标识：d 目录 | - 普通文件 | l 软链接

📄 二、 文件操作命令

命令 功能说明 注意事项
touch 创建空文件 / 更新已有文件时间戳 不修改文件内容，仅重置 atime 与 mtime
cp 复制文件或目录 cp -r src dest：递归复制目录（必加）
\cp src dest：绕过 alias cp='cp -i' 强制覆盖不提示
mv 移动或重命名 同盘移动极快（仅修改元数据）
mv old new：重命名
mv file /target/dir/：移动
rm 删除文件或目录 rm -f：强制删除不确认
rm -r：递归删除目录（逐层询问）
rm -rf：🔥 强制递归删除（不可逆）
rm -f ./*：清空当前目录（保留 . 本身）

⚠️ 高危警告：rm -rf 威力极大且无法撤销，执行前务必 pwd 确认当前路径，养成**“先查看，后执行”**的安全习惯。

👁️ 三、 文件内容查看与实时监控

Linux 查看文本内容有多种工具，按场景选择可大幅提升效率：

命令 适用场景 核心交互快捷键
cat 查看短小文件 / 拼接内容 cat -n 显示行号
cat f1 > f2 覆盖写入
cat f1 >> f2 追加写入
more 快速浏览中等长度日志（仅向下） Space 下翻一页 | Enter 下翻一行 | q 退出
less ⭐ 大文件翻阅 / 需双向搜索 Space/PgDn 下翻 | b/PgUp 上翻
/key 向下搜 | n 下一个 | N 上一个
G 跳尾 | g 跳头 | q 退出
head 查看文件头部 默认前 10 行
head -n 20 file 自定义行数
tail 查看文件尾部 / 动态监控 tail -n 20 file
tail -f file 🔥 实时追踪新增内容（排查日志必备）

💡 组合技巧：head -n 50 app.log | tail -n 10 可快速提取日志中间指定片段。

🔗 四、 软链接管理 (Symbolic Link)

软链接类似 Windows 的“快捷方式”，拥有独立的 inode，仅存储目标路径。

操作 命令示例 关键注意
创建 ln -s [源文件/目录] [链接名] 建议优先使用绝对路径，避免移动后失效
查看 ll 或 ls -l 首字符为 l，末尾显示 -> 真实路径
删除文件链接 rm link_name 仅删除链接，不影响源文件
删除目录链接 rm -rf link_name ⛔ 严禁在末尾加 /（如 rm -rf link/），否则会级联删除真实目录数据！

📡 五、 控制台输出与重定向 (echo)

echo 是最基础的终端输出命令，配合重定向符号可灵活操作文件内容。

⚠️ 重定向 > 具有破坏性，生产环境使用前建议先 ls 确认目标路径。

🕰️ 六、 历史命令管理 (history)

高效复用已执行命令，减少重复输入。

  • history：列出当前会话历史记录（带行号）
  • !<数字>：直接执行对应行号命令（例：!105）
  • history -c：清空当前终端历史
  • 🔄 效率进阶：
    • 按 Ctrl + R 进入交互式搜索模式，输入关键词快速定位，比 ! 更直观安全。
    • history \| grep "关键词" 精准筛选特定命令。

📊 核心命令速查对比表

实际需求 推荐命令 备注
查看短配置 / 代码 cat -n 直观带行号
快速浏览长日志（仅往下） more 简单轻量
仔细翻阅大文件 / 需搜索 less ⭐ 内存友好，功能最全
实时追踪日志更新 tail -f 运维排查标配
创建 / 更新时间戳 touch 不修改内容
复制目录 cp -r 注意 -r 位置
强制覆盖不提示 \cp 绕过 alias cp='cp -i'
彻底删除目录 rm -rf ⚠️ 执行前三思
移动或重命名 mv 同盘移动仅改元数据，极快

🛡️ 安全与最佳实践

  1. 破坏性操作前必做：pwd 确认路径 → ls 确认目标 → 再执行 rm/>。
  2. 善用帮助文档：man [命令] 或 [命令] --help 探索完整参数。
  3. 路径习惯：创建软链接或跨目录操作时，优先使用绝对路径，降低环境迁移导致的断裂风险。
  4. 肌肉记忆：Linux 命令重在实践。建议在本地虚拟机或 Docker 容器中反复练习，形成条件反射。
