📘 Linux 进程管理核心命令速查笔记

📋 1. ps：查看进程瞬时快照

ps 用于捕获执行瞬间的系统进程状态。由于进程数量庞大，实际使用中几乎总是搭配 grep 过滤。最常被问到的 ps aux 与 ps -ef 虽然都能列出全部进程，但底层设计流派与核心用途完全不同：

🔍 ps aux vs ps -ef 到底在做什么？

命令组合 设计流派 💡 核心作用（在做什么） 适用场景
ps aux BSD 风格 📊 侧重资源监控：直观展示每个进程的 CPU/内存消耗、运行状态与完整启动参数 快速定位“吃满资源”的进程、查看进程详细状态
ps -ef System V 风格 🌲 侧重进程血缘：突出父进程 ID（PPID），输出格式规整，便于脚本解析与链路追踪 排查服务启动链、定位僵尸进程源头、自动化脚本提取

📌 语法提示：严格来说 BSD 选项不带短横线（应为 ps aux），但 Linux 的 GNU ps 已做兼容，ps -aux 也可正常执行。

🔹 常用参数对照

参数 含义 备注
a 显示所有终端下的进程 含其他用户
u 以用户友好格式输出 显示 CPU/内存/状态等
x 显示无控制终端的后台进程 守护进程必备
-e 显示所有进程 等同于 a
-u 以用户格式输出 等同于 u
-f 全格式输出 包含 PPID、C 等字段

📖 输出字段速查

ps aux 常用字段

字段 含义
USER 进程所属用户
PID 进程 ID
%CPU CPU 使用率
%MEM 物理内存使用率
VSZ 虚拟内存大小（KB）
RSS 常驻物理内存大小（KB）
TTY 终端设备，? 表示无终端
STAT 进程状态（R运行 S睡眠 DIO等待 Z僵尸 T停止）
START 进程启动时间
TIME 累计占用 CPU 时间
COMMAND 完整启动命令及参数

ps -ef 常用字段

字段 含义
UID 用户 ID
PID 进程 ID
PPID 父进程 ID（排查依赖关系核心）
C CPU 使用率估算值
STIME 启动时间
TTY 终端
TIME CPU 累计时间
CMD 进程名（通常不含长参数）

💻 实战示例

🛑 2. kill & killall：安全终止进程

用于向指定进程发送控制信号，最常用于优雅关闭或强制清理卡死进程。

🔹 基础语法

🔹 常用信号对照

选项 信号名 编号 说明
(默认) SIGTERM 15 优雅退出，允许进程保存数据、清理资源
-9 SIGKILL 9 强制终止，内核直接回收，进程无法拦截

⚠️ 最佳实践：始终优先使用默认 kill <PID>。仅在进程完全无响应时，才使用 kill -9 <PID>。

🌳 3. pstree：进程依赖关系可视化

以树状结构展示进程的父子派生关系，直观呈现服务启动拓扑。

🔹 常用参数

参数 说明
-p 在进程名后显示 PID
-u 显示进程所属用户
-pu 同时显示 PID 与用户信息

📊 4. top：实时性能监控面板

动态刷新系统进程状态，是排查 CPU/内存瓶颈的首选交互工具。

🔹 启动参数

参数 说明
-d <秒> 设置屏幕刷新间隔（如 top -d 2）
-i 不显示空闲或僵尸进程
-p <PID> 仅监控指定进程

🔹 运行时交互快捷键

按键 功能
P 按 CPU 使用率 降序排序（默认）
M 按 内存使用率 降序排序
N 按 PID 排序
u 输入用户名，过滤仅显示该用户进程
k 输入 PID 与信号值，终止指定进程
q 退出 top

🌐 5. netstat：网络连接与端口占用排查

查看 TCP/UDP 连接状态、服务监听端口及对应进程归属。

🔹 高频实战组合

🔹 参数说明

参数 含义
-a 显示所有连接与监听端口
-n 以数字形式显示 IP 与端口（关闭 DNS 反向解析，提速）
-l 仅显示处于 LISTEN（监听）状态的端口
-p 显示占用端口的 PID 与程序名称

🔄 补充提示：部分新版 Linux 发行版已默认弃用 netstat，推荐使用等效工具 ss：
ss -tulnp | grep <端口号>

📎 场景速查表

排查目标 推荐命令
查进程资源占用 ps aux | grep <关键词>
查进程父子关系 ps -ef | grep <关键词>
优雅停止服务 kill <PID>
强制清理卡死进程 kill -9 <PID>
按名称批量杀进程 killall <名称>
查看服务依赖拓扑 pstree -pu
实时监控 CPU/内存 top → 按 P/M 切换排序
查端口归属进程 `netstat -nlp
