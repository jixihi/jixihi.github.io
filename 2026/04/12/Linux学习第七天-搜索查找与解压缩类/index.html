🐧 Linux 常用命令速查：搜索查找与压缩解压

📖 前言

在 Linux 日常操作中，快速定位文件与高效归档压缩是最基础也最高频的技能。本文基于实操笔记整理，按使用场景分类，配合参数表、代码示例与避坑指南，助你形成肌肉记忆，告别反复查阅。

🔍 一、搜索查找类命令

📊 核心命令对比速查

命令 查找目标 核心特点 典型场景
find 文件/目录（名称、大小、权限等） 实时全盘扫描，功能最强但稍慢 精确查找、多条件过滤
locate 文件路径 依赖本地数据库，速度极快 快速定位已知文件名
which 可执行程序 仅检索 $PATH 环境变量 确认命令安装路径
whereis 二进制/源码/man手册 范围固定，速度较快 查找命令关联文件位置
grep 文件内容 文本过滤神器，常配合管道使用 在日志/配置文件中查关键字

💡 核心使用详解

1. find：实时精准查找

常用选项：

  • -name：按文件名匹配（支持 * ? 通配符）
  • -user：按文件所属用户查找
  • -size：按文件大小查找（+ 大于，- 小于，k/M/G 为单位）

  • -n：显示匹配行及行号
  • |（管道符）：将前一个命令的标准输出，传递给后一个命令作为标准输入。

📊 压缩格式对比速查

命令 压缩格式 特点 适用场景
gzip / gunzip .gz 单文件压缩，Linux 原生，压缩率高 日志归档、单文件瘦身
zip / unzip .zip 跨平台兼容性好，支持目录递归 与 Windows/macOS 交互
tar .tar / .tar.gz 打包+压缩，完整保留权限与目录结构 服务器备份、源码分发

3. tar：打包解压神器（🌟 推荐掌握）

Linux 下最常用的归档工具，参数组合灵活。

核心参数口诀： cvfxzC

参数 含义 记忆技巧
-c create，创建归档 📦 打包
-v verbose，显示详细过程 👁️ 看进度
-x extract，解归档 📂 解压
-z gzip，调用 gzip 压缩/解压 🔧 处理 .gz
-f file，指定归档文件名 📛 必须放最后
-C change directory，指定解压路径 🎯 定向释放

🧠 三、实战经验总结

  1. 按需选查找命令：
    • 刚创建的文件 locate 搜不到？→ 执行 updatedb 或直接上 find
    • 找命令装在哪？→ which 或 whereis
    • 翻日志找报错？→ grep -n "ERROR" app.log | tail -20
  2. 管道符是效率灵魂：cmd1 | cmd2 | cmd3 能实现复杂流处理，例如 find /var/log -name "*.log" | xargs grep -l "timeout"
  3. 压缩格式选择策略：
    • 纯 Linux 服务器交互 → 优先 tar -zcvf
    • 需发给非 Linux 环境同事 → 用 zip -r
    • 仅压缩单个大文本/日志 → gzip 最轻量
