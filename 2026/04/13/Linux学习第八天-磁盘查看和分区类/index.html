Linux学习第八天-磁盘查看和分区类

Posted on 2026-04-13 In Linux学习

🖥️ Linux 磁盘管理命令全解析：从查看到分区，一文搞懂！

📖 核心概念速览（防懵指南）

在敲命令之前，先理清 Linux 管理磁盘的 3 个核心逻辑：

  1. 看空间：到底是谁占满了硬盘？（du 查文件，df 查分区）
  2. 看结构 & 挂载：硬盘在哪？怎么接到系统里？（lsblk、mount、fstab）
  3. 划地盘：新硬盘怎么用？（fdisk 分区）

🔍 一、磁盘空间查看：du vs df

这两个命令最容易搞混，记住一句话：du 向下看（文件/目录），df 向上看（文件系统/分区）。

1. du (Disk Usage) - 查看文件/目录实际占用

参数 含义 记忆技巧
-h --human-readable，以 K/M/G 显示 Human（人类可读）
-a --all，显示所有文件（含隐藏） All（全部）
-c --total，最后显示总计 Count（合计）
-s --summarize，只显示目录总和 Summary（摘要）
--max-depth=n 控制向下递归的目录层数 层数限制器

💡 关键区别：df 看的是文件系统层面的剩余空间；du 看的是文件层面的占用。两者偶尔不一致，通常是因为有文件被删除但进程仍占用（lsof | grep deleted）。

📂 二、设备查看与挂载管理

1. lsblk (List Block Devices) - 树状查看硬盘结构

最直观的“硬盘地图”命令，比 fdisk -l 更适合日常查看。

2. mount / umount - 手动挂载与卸载

Linux 没有“盘符”概念，所有设备必须挂载到某个空目录才能访问。

⚠️ 注意：卸载前必须退出该目录（cd ~），且不能有进程在使用该目录，否则会报 target is busy。

✂️ 三、磁盘分区工具：fdisk

fdisk 是传统的 MBR 分区表管理工具（适合 ≤2TB 磁盘）。

交互模式常用按键：

按键 功能 说明
m menu 查看帮助（忘记命令必按）
p print 打印当前分区表
n new 新建分区（p主分区 / e扩展分区）
d delete 删除分区
w write 保存并退出（真正生效）
q quit 不保存退出

🌟 进阶提示：现代服务器磁盘通常 >2TB，建议改用 parted 或 gdisk（GPT分区表）。分区完成后，记得用 partprobe 或重启让内核重新读取分区表，再格式化（mkfs.ext4 /dev/sdb1）和挂载。

📊 命令速查表（一键截图备用）

命令 核心作用 黄金组合 一句话人话翻译
du 查目录/文件大小 du -sh <路径> “这个文件夹到底吃了多少硬盘？”
df 查分区剩余空间 df -h “我的系统盘/数据盘还剩多少？”
lsblk 看硬盘分区树 lsblk -f “电脑插了几块盘？格式是啥？挂哪了？”
mount 临时挂载设备 mount /dev/sdb1 /mnt “把U盘/新硬盘‘插’到系统目录上”
fstab 永久挂载配置 vim /etc/fstab “告诉系统：开机自动把硬盘挂上来”
fdisk 磁盘分区管理 fdisk -l “给空白硬盘‘画格子’（分区）”

💡 给新手的 3 条避坑建议

  1. 先查后动：分区、格式化、改 fstab 前，先用 lsblk 和 df -h 确认设备名，千万别操作错磁盘（如 /dev/sda 通常是系统盘）。
  2. 卸载失败怎么办：umount: target is busy → 用 lsof +D /挂载点 或 fuser -m /挂载点 查看谁在占用，杀掉进程或 cd 出去再卸载。
  3. 交互命令别慌：进 fdisk 后如果不知道下一步按什么，直接敲 m 回车，官方帮助永远最靠谱。

📝 本篇笔记基于实际运维场景整理，建议配合虚拟机反复练习 du/df/lsblk 组合拳。磁盘操作无小事，养成 备份+确认 的习惯，你就是团队里最稳的仔！
