🐧 Linux 软件包管理指南：RPM 与 YUM 详解

📋 前言

在 Linux 系统运维中，软件包管理是每位开发者和管理员必须掌握的核心技能。本文基于学习笔记，系统梳理了 RPM 包管理工具与 YUM 仓库配置的核心命令与实用技巧，助你高效管理软件依赖，告别”依赖地狱”！

🔹 一、RPM 包管理基础

什么是 RPM？

RPM（Red Hat Package Manager）是 Red Hat 系列 Linux 发行版的官方打包安装工具，用于软件的安装、查询、验证、升级和卸载。

📦 常用命令速查表

功能 命令示例 说明
查询已安装包 rpm -qa 列出系统所有已安装的 RPM 包
模糊查询 rpm -qa | grep 软件包名 过滤查找特定软件
查看包详情 rpm -qi 软件包名称 显示版本、厂商、安装时间等元数据
卸载软件 rpm -e 软件包名 正常卸载（会检查依赖）
强制卸载 rpm -e --nodeps 软件包名 忽略依赖关系强制卸载 ⚠️
安装软件 rpm -ivh 软件包.rpm -i安装 -v详细输出 -h显示进度

💡 参数小贴士
--nodeps：跳过依赖检查（谨慎使用，可能导致系统不稳定）

🔹 二、YUM 仓库：自动化包管理神器

🌟 为什么选择 YUM？

YUM（Yellowdog Updater Modified）基于 RPM 构建，核心优势：

  • ✅ 自动解决依赖关系，一键安装所有前置包
  • ✅ 支持远程仓库，软件源集中管理
  • ✅ 批量更新/回滚，系统维护更高效

⚙️ YUM 常用命令参数

命令 功能说明 典型用法
install 安装软件包 yum install -y vim
update 更新软件/系统 yum update httpd
check-update 检查可更新包 yum check-update
remove / erase 卸载软件 yum remove nginx
list 列出可用/已安装包 yum list available | grep mysql
clean 清理缓存 yum clean all
deplist 查看依赖关系 yum deplist nginx

🎯 推荐选项：-y 自动确认所有提示，适合脚本自动化

📘 Linux 进程管理核心命令速查笔记

📋 1. ps：查看进程瞬时快照

ps 用于捕获执行瞬间的系统进程状态。由于进程数量庞大，实际使用中几乎总是搭配 grep 过滤。最常被问到的 ps aux 与 ps -ef 虽然都能列出全部进程，但底层设计流派与核心用途完全不同：

🔍 ps aux vs ps -ef 到底在做什么？

命令组合 设计流派 💡 核心作用（在做什么） 适用场景
ps aux BSD 风格 📊 侧重资源监控：直观展示每个进程的 CPU/内存消耗、运行状态与完整启动参数 快速定位“吃满资源”的进程、查看进程详细状态
ps -ef System V 风格 🌲 侧重进程血缘：突出父进程 ID（PPID），输出格式规整，便于脚本解析与链路追踪 排查服务启动链、定位僵尸进程源头、自动化脚本提取

📌 语法提示：严格来说 BSD 选项不带短横线（应为 ps aux），但 Linux 的 GNU ps 已做兼容，ps -aux 也可正常执行。

🔹 常用参数对照

参数 含义 备注
a 显示所有终端下的进程 含其他用户
u 以用户友好格式输出 显示 CPU/内存/状态等
x 显示无控制终端的后台进程 守护进程必备
-e 显示所有进程 等同于 a
-u 以用户格式输出 等同于 u
-f 全格式输出 包含 PPID、C 等字段

用于向指定进程发送控制信号，最常用于优雅关闭或强制清理卡死进程。

🔹 基础语法

🔹 常用信号对照

Linux学习第八天-磁盘查看和分区类

🖥️ Linux 磁盘管理命令全解析：从查看到分区，一文搞懂！

📖 核心概念速览（防懵指南）

在敲命令之前，先理清 Linux 管理磁盘的 3 个核心逻辑：

  1. 看空间：到底是谁占满了硬盘？（du 查文件，df 查分区）
  2. 看结构 & 挂载：硬盘在哪？怎么接到系统里？（lsblk、mount、fstab）
  3. 划地盘：新硬盘怎么用？（fdisk 分区）

🔍 一、磁盘空间查看：du vs df

这两个命令最容易搞混，记住一句话：du 向下看（文件/目录），df 向上看（文件系统/分区）。

-s --summarize，只显示目录总和 Summary（摘要）
--max-depth=n 控制向下递归的目录层数 层数限制器

参数 含义
-h 以易读格式显示容量

📌 实战示例

💡 关键区别：df 看的是文件系统层面的剩余空间；du 看的是文件层面的占用。两者偶尔不一致，通常是因为有文件被删除但进程仍占用（lsof | grep deleted）。

📂 二、设备查看与挂载管理

1. lsblk (List Block Devices) - 树状查看硬盘结构

最直观的“硬盘地图”命令，比 fdisk -l 更适合日常查看。

2. mount / umount - 手动挂载与卸载

Linux 没有“盘符”概念，所有设备必须挂载到某个空目录才能访问。

⚠️ 注意：卸载前必须退出该目录（cd ~），且不能有进程在使用该目录，否则会报 target is busy。

3. 🤖 开机自动挂载：/etc/fstab

✂️ 三、磁盘分区工具：fdisk

fdisk 是传统的 MBR 分区表管理工具（适合 ≤2TB 磁盘）。

交互模式常用按键：

按键 功能 说明
m menu 查看帮助（忘记命令必按）
p print 打印当前分区表
n new 新建分区（p主分区 / e扩展分区）

w write 保存并退出（真正生效）
q quit 不保存退出

🌟 进阶提示：现代服务器磁盘通常 >2TB，建议改用 parted 或 gdisk（GPT分区表）。分区完成后，记得用 partprobe 或重启让内核重新读取分区表，再格式化（mkfs.ext4 /dev/sdb1）和挂载。

📊 命令速查表（一键截图备用）

命令 核心作用 黄金组合 一句话人话翻译
du 查目录/文件大小 du -sh <路径> “这个文件夹到底吃了多少硬盘？”
df 查分区剩余空间 df -h “我的系统盘/数据盘还剩多少？”
lsblk 看硬盘分区树 lsblk -f “电脑插了几块盘？格式是啥？挂哪了？”
mount 临时挂载设备 mount /dev/sdb1 /mnt “把U盘/新硬盘‘插’到系统目录上”
fstab 永久挂载配置 vim /etc/fstab “告诉系统：开机自动把硬盘挂上来”
fdisk 磁盘分区管理 fdisk -l “给空白硬盘‘画格子’（分区）”

💡 给新手的 3 条避坑建议

  1. 先查后动：分区、格式化、改 fstab 前，先用 lsblk 和 df -h 确认设备名，千万别操作错磁盘（如 /dev/sda 通常是系统盘）。
  2. 卸载失败怎么办：umount: target is busy → 用 lsof +D /挂载点 或 fuser -m /挂载点 查看谁在占用，杀掉进程或 cd 出去再卸载。
  3. 交互命令别慌：进 fdisk 后如果不知道下一步按什么，直接敲 m 回车，官方帮助永远最靠谱。

📝 本篇笔记基于实际运维场景整理，建议配合虚拟机反复练习 du/df/lsblk 组合拳。磁盘操作无小事，养成 备份+确认 的习惯，你就是团队里最稳的仔！

🧠 三、实战经验总结

  1. 按需选查找命令：
    • 刚创建的文件 locate 搜不到？→ 执行 updatedb 或直接上 find
    • 找命令装在哪？→ which 或 whereis
    • 翻日志找报错？→ grep -n "ERROR" app.log | tail -20
  2. 管道符是效率灵魂：cmd1 | cmd2 | cmd3 能实现复杂流处理，例如 find /var/log -name "*.log" | xargs grep -l "timeout"

    • 纯 Linux 服务器交互 → 优先 tar -zcvf
    • 需发给非 Linux 环境同事 → 用 zip -r
    • 仅压缩单个大文本/日志 → gzip 最轻量

Linux学习第五天-时间日期类

Posted on 2026-04-10 In Linux学习

📅 Linux 时间与日期命令：date & cal

🕒 一、date 命令详解

date 是 GNU Coreutils 提供的核心时间工具，支持格式化输出、时间戳转换、相对时间计算以及系统时间设置。

🔹 1. 基础用法与格式化输出

语法：date [选项] [+格式字符串]

命令示例 功能说明 输出示例
date 显示当前系统日期与时间（默认格式） Fri Apr 10 15:30:22 CST 2026
date +%Y 显示 4 位年份 2026
date +%m 显示月份（01~12） 04
date +%d 显示日期（01~31） 10
date "+%Y-%m-%d %H:%M:%S" 自定义标准时间格式（推荐） 2026-04-10 15:30:22

🔹 2. 查看与转换时间戳

命令示例 功能说明
date +%s 获取当前 Unix 时间戳（秒级，自 1970-01-01 00:00:00 UTC 起）
date -d @1700000000 "+%Y-%m-%d %H:%M:%S" 将时间戳转换为可读时间

🔹 3. 显示相对时间（过去/未来）

使用 -d（或 --date）参数可计算非当前时间，支持自然语言表达式：

命令示例 功能说明
date -d "1 day ago" 显示昨天此时的时间
date -d "+1 day" 或 date -d "tomorrow" 显示明天此时的时间
date -d "next monday" 显示下周一日期
date -d "2 weeks ago" "+%Y-%m-%d" 仅显示两周前的日期

date -d '-1 days ago' 在 GNU date 中实际等价于 明天（-1 表示减去“1天前”）。建议统一使用 +N day / -N day 语法，语义更清晰。

🔹 4. 设置系统时间

命令示例 功能说明
sudo date -s "2026-04-10 15:30:00" 手动设置系统时间（需 root 权限）
sudo hwclock --systohc 将设置好的系统时间同步到硬件时钟（BIOS/RTC）

🛡️ 生产环境建议：服务器时间应通过 chronyd 或 ntpd 自动同步，尽量避免手动修改。

📅 二、cal 命令详解

cal（Calendar）用于快速查看日历，轻量且支持终端直接渲染。

命令示例 功能说明
cal 显示当前月份日历
cal -3 显示上个月、本月、下个月（共3个月）
cal -m 以 周一 作为每周的第一天（默认周日）
cal 2026 显示指定年份的全年日历
cal -y 显示当前年份的全年日历
cal 04 2026 显示指定年月的日历（格式：月 年）

📊 三、核心格式符速查表

在脚本或日志中格式化时间时，常用占位符如下：

格式符 含义 示例
%Y 4位年份 2026
%y 2位年份 26
%m 月份（01-12） 04
%d 日期（01-31） 10
%H 小时（24小时制，00-23） 15
%M 分钟（00-59） 30
%S 秒（00-60） 22
%s Unix 时间戳（秒） 1744270222
%u 星期几（1=周一, 7=周日） 5
%F 等价于 %Y-%m-%d 2026-04-10

🐧 Linux 核心命令速查手册：文件、目录与终端管理

📁 一、 目录操作命令

命令 功能说明 常用示例 & 备注
pwd 打印当前工作目录的绝对路径 pwd
cd 切换工作目录 cd ../xxx（进入上级目录的 xxx）
cd -（返回上一次所在目录）
ls 列出目录内容 ls：默认非隐藏
ls -a：包含 . 开头的隐藏文件
ls -l：长格式详情（等效别名 ll）
ls -al：完整显示所有文件详情
mkdir 创建目录 mkdir dir
mkdir -p a/b/c：递归创建多级目录
rmdir 删除空目录 rmdir dir
rmdir -p a/b/c：递归删除空父目录链（遇非空即停止）

📌 ls -l 输出字段解析（从左至右）：
权限 硬链接数 所有者 所属组 文件大小 最后修改时间 名称
首字符标识：d 目录 | - 普通文件 | l 软链接

📄 二、 文件操作命令

命令 功能说明 注意事项
touch 创建空文件 / 更新已有文件时间戳 不修改文件内容，仅重置 atime 与 mtime
cp 复制文件或目录 cp -r src dest：递归复制目录（必加）
\cp src dest：绕过 alias cp='cp -i' 强制覆盖不提示
mv 移动或重命名 同盘移动极快（仅修改元数据）
mv old new：重命名
mv file /target/dir/：移动
rm 删除文件或目录 rm -f：强制删除不确认
rm -r：递归删除目录（逐层询问）
rm -rf：🔥 强制递归删除（不可逆）
rm -f ./*：清空当前目录（保留 . 本身）

⚠️ 高危警告：rm -rf 威力极大且无法撤销，执行前务必 pwd 确认当前路径，养成**“先查看，后执行”**的安全习惯。

👁️ 三、 文件内容查看与实时监控

Linux 查看文本内容有多种工具，按场景选择可大幅提升效率：

命令 适用场景 核心交互快捷键
cat 查看短小文件 / 拼接内容 cat -n 显示行号
cat f1 > f2 覆盖写入
cat f1 >> f2 追加写入
more 快速浏览中等长度日志（仅向下） Space 下翻一页 | Enter 下翻一行 | q 退出
less ⭐ 大文件翻阅 / 需双向搜索 Space/PgDn 下翻 | b/PgUp 上翻
/key 向下搜 | n 下一个 | N 上一个
G 跳尾 | g 跳头 | q 退出
head 查看文件头部 默认前 10 行
head -n 20 file 自定义行数
tail 查看文件尾部 / 动态监控 tail -n 20 file
tail -f file 🔥 实时追踪新增内容（排查日志必备）

💡 组合技巧：head -n 50 app.log | tail -n 10 可快速提取日志中间指定片段。

🔗 四、 软链接管理 (Symbolic Link)

软链接类似 Windows 的“快捷方式”，拥有独立的 inode，仅存储目标路径。

操作 命令示例 关键注意
创建 ln -s [源文件/目录] [链接名] 建议优先使用绝对路径，避免移动后失效
查看 ll 或 ls -l 首字符为 l，末尾显示 -> 真实路径
删除文件链接 rm link_name 仅删除链接，不影响源文件
删除目录链接 rm -rf link_name ⛔ 严禁在末尾加 /（如 rm -rf link/），否则会级联删除真实目录数据！

📡 五、 控制台输出与重定向 (echo)

echo 是最基础的终端输出命令，配合重定向符号可灵活操作文件内容。

⚠️ 重定向 > 具有破坏性，生产环境使用前建议先 ls 确认目标路径。

🕰️ 六、 历史命令管理 (history)

高效复用已执行命令，减少重复输入。

  • history：列出当前会话历史记录（带行号）
  • !<数字>：直接执行对应行号命令（例：!105）
  • history -c：清空当前终端历史
  • 🔄 效率进阶：
    • 按 Ctrl + R 进入交互式搜索模式，输入关键词快速定位，比 ! 更直观安全。
    • history \| grep "关键词" 精准筛选特定命令。
