Linux学习第四天-文件目录类命令

Posted on 2026-04-09 Edited on 2026-04-10 In Linux学习

🐧 Linux 核心命令速查手册：文件、目录与终端管理

📁 一、 目录操作命令

命令 功能说明 常用示例 & 备注
pwd 打印当前工作目录的绝对路径 pwd
cd 切换工作目录 cd ../xxx（进入上级目录的 xxx）
cd -（返回上一次所在目录）
ls 列出目录内容 ls：默认非隐藏
ls -a：包含 . 开头的隐藏文件
ls -l：长格式详情（等效别名 ll）
ls -al：完整显示所有文件详情
mkdir 创建目录 mkdir dir
mkdir -p a/b/c：递归创建多级目录
rmdir 删除空目录 rmdir dir
rmdir -p a/b/c：递归删除空父目录链（遇非空即停止）

📌 ls -l 输出字段解析（从左至右）：
权限 硬链接数 所有者 所属组 文件大小 最后修改时间 名称
首字符标识：d 目录 | - 普通文件 | l 软链接

📄 二、 文件操作命令

命令 功能说明 注意事项
touch 创建空文件 / 更新已有文件时间戳 不修改文件内容，仅重置 atime 与 mtime
cp 复制文件或目录 cp -r src dest：递归复制目录（必加）
\cp src dest：绕过 alias cp='cp -i' 强制覆盖不提示
mv 移动或重命名 同盘移动极快（仅修改元数据）
mv old new：重命名
mv file /target/dir/：移动
rm 删除文件或目录 rm -f：强制删除不确认
rm -r：递归删除目录（逐层询问）
rm -rf：🔥 强制递归删除（不可逆）
rm -f ./*：清空当前目录（保留 . 本身）

⚠️ 高危警告：rm -rf 威力极大且无法撤销，执行前务必 pwd 确认当前路径，养成**“先查看，后执行”**的安全习惯。

👁️ 三、 文件内容查看与实时监控

Linux 查看文本内容有多种工具，按场景选择可大幅提升效率：

命令 适用场景 核心交互快捷键
cat 查看短小文件 / 拼接内容 cat -n 显示行号
cat f1 > f2 覆盖写入
cat f1 >> f2 追加写入
more 快速浏览中等长度日志（仅向下） Space 下翻一页 | Enter 下翻一行 | q 退出
less ⭐ 大文件翻阅 / 需双向搜索 Space/PgDn 下翻 | b/PgUp 上翻
/key 向下搜 | n 下一个 | N 上一个
G 跳尾 | g 跳头 | q 退出
head 查看文件头部 默认前 10 行
head -n 20 file 自定义行数
tail 查看文件尾部 / 动态监控 tail -n 20 file
tail -f file 🔥 实时追踪新增内容（排查日志必备）

💡 组合技巧：head -n 50 app.log | tail -n 10 可快速提取日志中间指定片段。

🔗 四、 软链接管理 (Symbolic Link)

软链接类似 Windows 的“快捷方式”，拥有独立的 inode，仅存储目标路径。

操作 命令示例 关键注意
创建 ln -s [源文件/目录] [链接名] 建议优先使用绝对路径，避免移动后失效
查看 ll 或 ls -l 首字符为 l，末尾显示 -> 真实路径
删除文件链接 rm link_name 仅删除链接，不影响源文件
删除目录链接 rm -rf link_name ⛔ 严禁在末尾加 /（如 rm -rf link/），否则会级联删除真实目录数据！

📡 五、 控制台输出与重定向 (echo)

echo 是最基础的终端输出命令，配合重定向符号可灵活操作文件内容。

⚠️ 重定向 > 具有破坏性，生产环境使用前建议先 ls 确认目标路径。

🕰️ 六、 历史命令管理 (history)

高效复用已执行命令，减少重复输入。

  • history：列出当前会话历史记录（带行号）
  • !<数字>：直接执行对应行号命令（例：!105）
  • history -c：清空当前终端历史
  • 🔄 效率进阶：
    • 按 Ctrl + R 进入交互式搜索模式，输入关键词快速定位，比 ! 更直观安全。
    • history \| grep "关键词" 精准筛选特定命令。

📊 核心命令速查对比表

实际需求 推荐命令 备注
查看短配置 / 代码 cat -n 直观带行号
快速浏览长日志（仅往下） more 简单轻量
仔细翻阅大文件 / 需搜索 less ⭐ 内存友好，功能最全
实时追踪日志更新 tail -f 运维排查标配
创建 / 更新时间戳 touch 不修改内容
复制目录 cp -r 注意 -r 位置
强制覆盖不提示 \cp 绕过 alias cp='cp -i'
彻底删除目录 rm -rf ⚠️ 执行前三思
移动或重命名 mv 同盘移动仅改元数据，极快

🛡️ 安全与最佳实践

  1. 破坏性操作前必做：pwd 确认路径 → ls 确认目标 → 再执行 rm/>。
  2. 善用帮助文档：man [命令] 或 [命令] --help 探索完整参数。
  3. 路径习惯：创建软链接或跨目录操作时，优先使用绝对路径，降低环境迁移导致的断裂风险。
  4. 肌肉记忆：Linux 命令重在实践。建议在本地虚拟机或 Docker 容器中反复练习，形成条件反射。

📝 Vim 编辑器核心快捷键笔记：三大模式速查指南

🟦 一、普通模式（Normal Mode）

Vim 启动后的默认模式。光标移动、文本复制/删除、撤销等操作均在此完成。

快捷键 功能说明 🛠️ 规范与实战提示
y 复制（Yank） 需配合移动指令使用，如 yw
yy 复制当前行
ny 复制 n 行 实际更常用标准写法 nyy
yw 复制一个词 从光标处复制到词尾
yny 复制含光标的 n 行
y$ 复制光标后至行尾 包含当前字符
y^ 复制光标前至行首 不包含行首缩进（可用 y0 包含）
p 粘贴 粘贴至光标后方
np 重复粘贴 n 次 如 3p 粘贴三次
d 删除/剪切 删除内容会进入寄存器，可用 p 粘贴
dw 删除一个词
d$ / d^ 删除至行尾 / 行首
nd / dnd 删除 n 行 推荐标准写法：ndd
u 撤销（Undo） 可连续按 Ctrl+r 重做（Redo）
x 删除光标处字符 笔记记为“剪贴一个词”，实为单字符删除
g / G / nG 回到文本开始 / 末尾 / 第n行 Vim 标准首行跳转指令为 gg
w / e 跳至下一个词首 / 词尾 可搭配数字，如 3w 跳3个词
set nu / set nonu 显示行号 / 撤销行号 需在命令模式下输入 : 执行

💡 核心语法：[数量] + [操作符(y/d/c)] + [移动指令(w/$/^/G)]
例如：3dw = 删除3个词 ｜ 2y$ = 复制2次光标至行尾

🟩 二、编辑模式（Insert Mode）

用于实际输入文本。按 Esc 可随时退回普通模式。

快捷键 功能说明 🎯 适用场景
i 在光标前插入 精准修改当前字符前方内容
a 在光标后插入 适合在当前字符后追加文本
o 在光标下一行新建并插入 自动换行并继承缩进，写代码最常用

🔑 记忆口诀：i = Insert（插入）｜ a = Append（追加）｜ o = Open new line（新开一行）

🟥 三、命令模式（Command / Ex Mode）

在普通模式下输入 : 进入底行，用于文件操作、查找替换、环境配置。

命令 功能说明 🔍 细节说明
/word 查找单词 word 按 n 向下匹配，N 向上匹配
:s/old/new 替换当前行第一个匹配项 仅作用于光标所在行
:s/old/new/g 替换当前行所有匹配项 g = global（本行全局）
:%s/old/new 替换全文第一个匹配项 % 代表整个文件范围
:%s/old/new/g 替换全文所有匹配项 最常用批量替换命令
noh 取消搜索高亮 实际执行需加前缀：:noh
w 保存文件 实际执行：:w（Write）
wq 保存并退出 实际执行：:wq
q! 强制退出（不保存） 实际执行：:q!

⚠️ 交互替换技巧：在替换命令末尾添加 c（如 :%s/old/new/gc）可开启逐次确认模式，避免误替换重要代码。

🐧 Linux 根目录结构详解

🌲 目录结构全景

Linux 文件系统以 / 为根，采用严格的树状层级规范。以下是核心目录的精准定位与功能说明：

目录路径 英文全称 / 含义 核心作用与说明
/bin Binary 存放所有用户可执行的基础命令（如 ls、cp、cat、bash）
/sbin System Binary 存放系统管理员专用的管理命令（如 fdisk、iptables、systemctl）
/lib /lib64 Library 系统启动与程序运行所需的共享库文件（.so）及动态链接器；lib64 专用于 64 位架构
/usr Unix System Resources 存放用户级应用程序、库、文档与头文件（⚠️ 非 user 缩写，是系统中最大的只读数据目录）
/usr/local Local Software 本地编译安装的第三方软件独立目录，避免与系统包管理器（apt/yum）冲突
/boot Boot Loader 系统启动核心文件：Linux 内核（vmlinuz）、初始内存盘（initramfs）、GRUB 引导配置
/dev Device 设备文件目录，代表所有硬件（如 /dev/sda 硬盘、/dev/tty 终端），由 udev 动态管理
/etc Editable Text Configuration 系统及应用的全局配置文件（如 /etc/passwd、/etc/ssh/sshd_config），通常为纯文本
/home Home Directories 普通用户主目录，存放个人文档、配置与数据（如 /home/alice）
/root Root Home 超级管理员专属主目录，独立于 /home，权限严格隔离
/opt Optional 第三方商业/大型软件安装目录（如 Oracle、MATLAB、Chrome），通常自包含、不干扰系统
/media Media 系统自动挂载可移动介质（U盘、光盘、SD卡）的默认入口
/mnt Mount 供管理员手动临时挂载文件系统的通用目录（如测试分区、NFS/CIFS 共享）
/proc Process 虚拟文件系统（内存映射），实时反映进程状态与内核参数（如 /proc/cpuinfo、/proc/meminfo）
/run Runtime 存放系统运行时的临时状态数据（PID 文件、socket、锁文件），基于 tmpfs，重启清空
/srv Service 本机对外提供的服务数据目录（如 Web 服务器 /srv/www、FTP 服务器 /srv/ftp）
/sys System 虚拟文件系统，提供内核与硬件/驱动/电源管理的交互接口（常用于设备调试与电源策略）
/tmp Tmporary 全局临时目录，任意用户可读写，通常重启自动清理或配置 tmpwatch 定期轮转
/var Variable 存放频繁变更的数据：日志（/var/log）、缓存（/var/cache）、邮件队列、数据库文件等

🔍 关键修正与避坑指南

  • ✅ user → 实际为 /usr。历史命名源于早期 Unix 将用户工具与系统核心分离，并非 user 的缩写。
  • ✅ ect → 正确拼写为 /etc。早期用于存放 “et cetera”（杂项）文件，现代 Linux 已演进为配置中心。
  • 🚫 勿随意修改 /proc 与 /sys 中的节点，除非明确知晓对应内核接口作用。
  • 🗑️ /tmp 和 /run 属于易失性目录，重要业务数据与持久化配置切勿存放于此。
  • 📦 安装源码或第三方软件优先选择 /opt 或 /usr/local，保持系统目录纯净，便于升级与卸载。
  • 🔐 修改 /etc 配置前务必备份：sudo cp -a /etc/ssh/sshd_config /etc/ssh/sshd_config.bak

🧠 高效记忆口诀

📦 bin/sbin 放命令，lib/lib64 存库文件；
🌐 usr 装资源，boot 管启动；
🔧 dev 映射硬件，etc 管配置；
🏠 home 住用户，root 独居处；
📥 opt 装第三方，media/mnt 挂载区；
🧵 proc 看进程，sys 调驱动；
🕰️ run 存状态，tmp 临时用，var 记日志。
